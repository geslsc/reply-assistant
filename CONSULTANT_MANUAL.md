# 客立樂教學小助手｜顧問操作手冊

本文件說明顧問可對小助手下達的指令、自然語法範例、小助手動作、是否公開回群組，以及是否需要二次確認。

---

## 自我介紹

| 項目 | 內容 |
|------|------|
| 顧問想做的事 | 在群組向店家介紹小助手 |
| 建議標準語法 | `小助手自我介紹一下` |
| 可接受自然語法範例 | 「自我介紹一下」「跟店家介紹小助手」 |
| 小助手動作 | 發送固定自我介紹文案至群組 |
| 是否公開回群組 | 是 |
| 是否需要確認 | 否 |

## 代回群組

| 項目 | 內容 |
|------|------|
| 顧問想做的事 | 將顧問私訊內容逐字代回群組 |
| 建議標準語法 | 私訊 `代回群組：[內容]` 或 `Q-YYYYMMDD-HHMM-XX [內容]` |
| 可接受自然語法範例 | 「回覆這題：…」「幫我回群組：…」 |
| 小助手動作 | 通過三道檢查後，以 pushMessage 逐字轉貼至群組 |
| 是否公開回群組 | 是 |
| 是否需要確認 | **是**（高副作用；多 pending 時須指定短碼） |

代回前三道檢查：
1. 是哪一題（多 pending 須指定短碼；單筆可用「這題」）
2. thread 是否仍 open（已結案 / 群組 mute / OUT_OF_SERVICE_PERIOD / 服務期結束則拒絕；第一版不做 thread mute）
3. 權限（限 active admin 或 consultant）

## 啟用保母期 30 天

| 項目 | 內容 |
|------|------|
| 顧問想做的事 | 啟用 30 天教學協助期 |
| 建議標準語法 | **僅允許** `小助手啟用保母期 30 天` 或 `小助手開始協助 30 天` |
| 可接受自然語法範例 | **不開放**（避免閒聊誤觸發 serviceStartAt / serviceEndAt） |
| 小助手動作 | 寫入 serviceStartAt / serviceEndAt 並發送自我介紹 |
| 是否公開回群組 | 是 |
| 是否需要確認 | 否 |

---

## 附註

- 知識卡草稿不會自動生效；正式知識庫仍由管理者手動貼 JSON 並 commit。
- REPLY_TO_GROUP 一律逐字轉貼，LLM 不得改寫顧問內容。
- 代回群組事件記錄於 event_logs，event_type 沿用 `consultant_override`，detail 含 `intent=REPLY_TO_GROUP`。
